GitHub Access Token
-------------------
The access token can be used instead of SSH when accessing GitHub from the command line with "gh" and/or "git".

To get a GitHub token go to your GitHub account --> Settings --> Developer Settings --> Personal Access Tokens --> Tokens (classic) --> Generate New Token --> Generate New Token (classic)
	- fill in the note (eg.: "Temporary Token for <name> Through <date>")
	- select 30 days or whatever
	- select scopes:
			repo
			read:org
			workflow
			delete_repo
--> Generate Token
- copy the token and save it to your passwords file

gh auth login

Create New Repo From Command Line
---------------------------------
* We can no longer create remote repositories on GitHub from the terminal
  / command line using `git` alone.  We must now use the `gh` GitHub
  command line interface tool to create and delete remote repositories on
  GitHub.

	mkdir <reponame>
	cd <reponame>
	git init
	git branch -M master
	echo "# <reponame>" >> README.md
	git add README.md
	git commit -m "initial commit"
	gh auth login
	gh repo create <reponame> --public --source=. --remote=origin
	# git remote add origin https://github.com/<username>/<reponame>
	git push -u origin master

** "Username" is your GitHub username, "Password" is your GitHub token.
	When you have pushed your first commit and created the repo, you
	can create a "fine-grained" token or SSH key strictly for that
	repository.


Push An Existing Repo From Command Line To An Existing Repo On GitHub
---------------------------------------------------------------------
git remote add origin https://github.com/<username>/<reponame>.git
git branch -M master
git push -u origin master

** "Username" is your GitHub username, "Password" is your GitHub token.
